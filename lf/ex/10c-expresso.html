1. A conquista Hindley-Milner

Hindley-Milner une segurança de tipos, inferência automática e polimorfismo paramétrico. Hindley descreveu a base teórica; Milner a tornou prática em ML; Damas formalizou as provas.

Promessa

Se uma expressão tem tipo no sistema, o algoritmo encontra seu tipo mais geral.

2. Tipos e esquemas

Tipos incluem variáveis, tipos base e funções. Esquemas introduzem quantificação universal para representar polimorfismo.

$τ ::= α | T | τ₁ → τ₂$
$σ ::= τ | ∀α.σ$

O exemplo clássico é a identidade: $id : ∀α. α → α$.

3. Julgamentos de tipo

O contexto $Γ$ associa nomes a esquemas de tipo. Regras como variável, abstração, aplicação e `let` determinam quando uma expressão é bem tipada.

4. O coração: unificação

Unificar é encontrar substituições para variáveis de tipo que tornam dois tipos iguais. O occur check impede tipos infinitos como $α = α → int$.

unify(α, int) = {α ↦ int} unify(α → α, bool → β) = {α ↦ bool, β ↦ bool} unify(α, α → int) = erro de occur check

5. Algoritmo W e limites

O Algoritmo W percorre a AST, gera substituições por unificação, instancia esquemas ao usar nomes e generaliza tipos em `let`. Ele é correto, completo e produz tipos principais para o núcleo HM.

Ponto forte

Inferência completa para o núcleo funcional com polimorfismo paramétrico.

Custo prático

Unificação e tipos grandes podem tornar mensagens de erro difíceis.

Extensões modernas

Type classes, GADTs, subtipagem e higher-rank types exigem restrições ou anotações.
