1. Introdução à Análise Semântica

"Colorless green ideas sleep furiously" - Noam Chomsky

Uma frase sintaticamente perfeita, mas semanticamente absurda

Após as fases de análise léxica e sintática, um compilador já será capaz de reconhecer a estrutura de um programa. Contudo, a análise sintática é agnóstica ao significado. A análise semântica é a fase que preenche essa lacuna, verificando se as construções sintaticamente válidas formam um conjunto de instruções coerente e com significado.

Declarações

Foram y e z declarados anteriormente?

Compatibilidade

Seus tipos são compatíveis com o operador +?

Armazenamento

O resultado pode ser armazenado em uma variável int?

Contra-intuições Semânticas

Python

# Multiplicação de string por booleano
resultado = "texto" * True  # "texto"
resultado = "texto" * False  # ""

# Soma de booleanos
valor = True + True + False  # 2

# Lista multiplicada por negativo
lista = [1, 2, 3] * -1  # []

JavaScript

// NaN não é igual a si mesmo
NaN === NaN;  // false

// Conversão implícita
"5" - 3;  // 2
"5" + 3;  // "53"

// Array com buracos
const arr = [1,,3];
arr.length;  // 3

2. Perspectiva Histórica

A análise semântica não nasceu pronta. Sua história é uma jornada que acompanha a própria evolução das linguagens de programação, partindo de verificações simples e dispersas para se tornar uma estrutura formal e sofisticada.

1957 - FORTRAN I

Análise semântica ad-hoc, verificações entrelaçadas com geração de código

1960 - ALGOL 60

Primeira separação clara entre sintaxe e semântica, introdução de escopo léxico

1968 - Donald Knuth

Gramáticas de Atributos - formalismo sistemático para análise semântica

1978 - Robin Milner

Algoritmo W para inferência de tipos, base do sistema de tipos ML

Marcos Fundamentais

Formalização

Floyd (1967) - Semântica com lógica formal através de asserções

Sistematização

Aho & Ullman (1977) - Livro do Dragão estabelece estrutura canônica

Complexidade

C++ (1985) - Templates e sobrecarga exigem análise complexa

3. Da Necessidade à Solução: Por Que Gramáticas de Atributos?

O Problema Fundamental

As gramáticas livres de contexto (GLC), poderosas para descrever a sintaxe, revelavam-se insuficientes para capturar regras semânticas.

Limitação das GLCs: Uma GLC pode garantir que x = y + z; é sintaticamente válida, mas não pode verificar se y e z foram declarados.

Problemas do Código Ad-hoc

  • Não-portabilidade
  • Verificação impossível
  • Manutenção custosa
  • Falta de clareza

Solução de Knuth (1968)

  • Estende naturalmente as GLCs
  • Fluxo de informação contextual
  • Implementação sistemática
  • Base para especificação

A Ideia Central

Se as produções da gramática descrevem a estrutura do programa, por que não anotar essas mesmas produções com regras que descrevem o significado?

1. Atributos: Propriedades associadas aos símbolos gramaticais
2. Regras Semânticas: Equações que definem como calcular atributos

5. Semântica Estática vs. Dinâmica

As regras semânticas de uma linguagem se dividem em duas categorias, que definem o que pode e o que não pode ser verificado antes da execução do programa.

Semântica Estática

Regras verificadas em tempo de compilação, através da análise do código-fonte.

Escopo e Declarações
  • Variáveis declaradas
  • Identificadores únicos
  • Ocultação de escopo
Verificações de Tipos
  • Compatibilidade de operadores
  • Tipos em atribuições
  • Expressões condicionais
Controle de Fluxo
  • break/continue em loops
  • return em funções
  • Labels de goto

Semântica Dinâmica

Aspectos que só podem ser determinados em tempo de execução.

Erros Aritméticos
int divisor = obterValor();
int resultado = 100 / divisor;  // Erro se divisor == 0
Acesso à Memória
int[] nums = new int[10];
int indice = calcularIndice();
int valor = nums[indice];  // Erro se indice < 0 ou >= 10
Indecidibilidade: O Problema da Parada de Turing prova que não existe algoritmo geral que possa determinar todos os erros de execução estaticamente.

6. Gramáticas de Atributos

Definição Formal

Uma Gramática de Atributos (AG) é uma tupla $AG = \{G, A, R\}$, onde:

  • $G$: Gramática Livre de Contexto $(N, T, P, S)$
  • $A$: Conjunto de Atributos $= \bigcup_{X \in N \cup T} A(X)$
  • $R$: Conjunto de Regras Semânticas

Atributos Sintetizados

Valor calculado a partir dos filhos. Informação flui de baixo para cima.

expr.valor = expr1.valor + term.valor

Atributos Herdados

Valor calculado a partir do pai ou irmãos. Informação flui de cima para baixo.

lista_vars.tipo = tipo.tipo_sintetizado

Exemplo: Declaração de Variáveis

Para processar float x, y;, o tipo deve ser distribuído para cada variável.

Tipo → float Tipo.tipo = 'real'
ListaIDs → ListaIDs₁, id ListaIDs₁.tipo_herdado = ListaIDs.tipo_herdado
adicionar_tipo(id.lexval, ListaIDs.tipo_herdado)

7. Verificação de Tipos

A verificação de tipos é a tarefa central da análise semântica, garantindo que os tipos de dados sejam usados de maneira consistente e segura.

Tipagem Estática

Verificação em tempo de compilação

Java C++ Rust Haskell
  • Detecção precoce de erros
  • Performance otimizada
  • Menos flexível

Tipagem Dinâmica

Verificação em tempo de execução

Python JavaScript Ruby LISP
  • Flexibilidade
  • Menos verboso
  • Erros em runtime

Julgamento de Tipo

$$\Gamma \vdash e : \tau$$

"No contexto Γ, é possível provar que a expressão e tem o tipo τ"

Γ (Gama)
Contexto/Tabela de Símbolos
⊢ (Turnstile)
Símbolo de derivação
e : τ
Expressão tem tipo τ

Regras de Inferência de Tipos

Regra da Variável
$$\frac{x : \tau \in \Gamma}{\Gamma \vdash x : \tau}$$
Regra da Aplicação
$$\frac{\Gamma \vdash e_1 : \tau_1 \rightarrow \tau_2 \quad \Gamma \vdash e_2 : \tau_1}{\Gamma \vdash e_1 \, e_2 : \tau_2}$$
Regra do Operador
$$\frac{\Gamma \vdash e_1 : \text{int} \quad \Gamma \vdash e_2 : \text{int}}{\Gamma \vdash e_1 + e_2 : \text{int}}$$

8. Implementação Prática: Mini-Linguagem Langarit

Aplicaremos os conceitos construindo um analisador semântico completo para uma linguagem de expressões aritméticas.

Características de Langarit

Suportado
  • Declarações de variáveis (int, float)
  • Expressões aritméticas (+, -, *, /)
  • Atribuições de valores
  • Literais numéricos
Erros Detectados
  • Variável não declarada
  • Atribuição de float a int
  • Divisão por zero literal
  • Redeclaração de variável

Gramática de Langarit

$$\begin{align*} \text{Programa} & \rightarrow \text{Declarações} \; \text{Comandos} \\ \text{Declaração} & \rightarrow \text{Tipo} \; \textbf{id} \; \textbf{;} \\ \text{Tipo} & \rightarrow \textbf{int} \mid \textbf{float} \\ \text{Comando} & \rightarrow \textbf{id} \; \textbf{=} \; \text{Expr} \; \textbf{;} \\ \text{Expr} & \rightarrow \text{Expr} \; \textbf{+} \; \text{Term} \mid \text{Term} \\ \text{Term} & \rightarrow \text{Term} \; \textbf{*} \; \text{Factor} \mid \text{Factor} \\ \text{Factor} & \rightarrow \textbf{id} \mid \textbf{num} \end{align*}$$

Exemplo de Implementação (Python) 

class AnalisadorSemantico:
    def __init__(self):
        self.tabela = TabelaSimbolos()
        self.erros = []
    
    def _analisar_declaracao(self, decl):
        """Adiciona variável à tabela de símbolos"""
        try:
            self.tabela.adicionar(decl.nome, decl.tipo)
        except Exception as e:
            self.erros.append(str(e))
    
    def _analisar_atribuicao(self, atrib):
        """Verifica compatibilidade de tipos"""
        tipo_var = self.tabela.consultar(atrib.variavel)
        if tipo_var is None:
            self.erros.append(f"Variável '{atrib.variavel}' não declarada")
            return
        
        tipo_expr = self._analisar_expressao(atrib.expressao)
        
        # Verificar compatibilidade
        if tipo_var == "int" and tipo_expr == "float":
            self.erros.append(f"Não é possível atribuir float a int")
    
    def _promover_tipo(self, tipo1, tipo2):
        """Regras de promoção de tipos"""
        if tipo1 == "float" or tipo2 == "float":
            return "float"
        return "int"
Programa Válido
int x;
float y;
x = 10;
y = 3.14;
x = x + 5;
y = y * 2.0;
Programa com Erros
x = 10;        # x não declarado
int x;
x = 3.14;      # float para int
int y;
y = x / 0;     # divisão por zero

9. A Ponte para a Geração de Código Intermediário

A análise semântica marca o fim da fase de análise do compilador. A AST, agora ricamente atribuída com informações de tipo e escopo, está pronta para ser entregue à fase de síntese.

Código de Três Endereços (TAC)

Código Fonte
w = (a + b) * c;
Código TAC Gerado
t1 = a + b
t2 = t1 * c
w = t2

Processo de Geração

1
Travessia da AST: Percorremos a árvore recursivamente
2
Geração de temporárias: Criamos variáveis t1, t2 para resultados intermediários
3
Emissão de instruções: Geramos instruções de três endereços
4
Linearização: Transformamos estrutura hierárquica em sequência linear
Modularidade

Separa front-end do back-end

Portabilidade

Facilita suporte a múltiplas arquiteturas

Otimização

Base para otimizações independentes

10. Exercícios

Exercício 1: Semântica Básica

Dado o programa:

x := 3;
y := x * 2;
x := x + y;

Trace a execução usando semântica denotacional.

Solução:
  • $\sigma_0 = \{\}$ (estado inicial)
  • $\sigma_1 = \{x \mapsto 3\}$
  • $\sigma_2 = \{x \mapsto 3, y \mapsto 6\}$
  • $\sigma_3 = \{x \mapsto 9, y \mapsto 6\}$
Exercício 2: Análise de Tipos

Verifique se o programa está bem tipado:

int x = 5;
bool b = (x > 0);
int y = if b then x + 1 else x - 1;
Solução:

Programa bem tipado!

  1. $x : \text{int}$
  2. $x > 0 : \text{bool}$ (comparação de inteiros)
  3. $b : \text{bool}$
  4. $x + 1 : \text{int}$
  5. $x - 1 : \text{int}$
  6. $\text{if } b \text{ then int else int} : \text{int}$
  7. $y : \text{int}$
Exercício 3: Otimização

Identifique otimizações possíveis:

x := 10;
y := x + 0;
z := y * 1;
w := z - z;
Solução:

Aplicando regras semânticas de identidade:

x := 10;
y := x;    # x + 0 → x
z := y;    # y * 1 → y
w := 0;    # z - z → 0
Exercícios Propostos para Extensão
  1. Operadores Relacionais: Adicione suporte para <, >, <=, >=, ==, !=
  2. Comandos Condicionais: Implemente if-else com verificação de tipo booleano
  3. Múltiplas Declarações: Suporte int x, y, z;
  4. Verificação de Inicialização: Detecte uso de variáveis não inicializadas
  5. Funções e Escopo: Adicione funções com parâmetros e variáveis locais
  6. Mensagens de Erro: Melhore com números de linha e sugestões
