18  MLIR 101: Primeiro Exemplo e Sintaxe Básica

Tudo no MLIR gira em torno de alguns conceitos principais.

18.0.1 Dialetos (Dialects)

Este é o conceito mais importante. O MLIR por si só não define quase nada. A “linguagem” que você usa é, na verdade, um Dialeto.

  • Um Dialeto é um conjunto de operações, tipos e atributos com uma finalidade específica.
  • Exemplos de Dialetos:
    • func: Para definir funções (ex: func.func @main...).
    • arith: Para operações aritméticas (ex: arith.constant, arith.addi).
    • affine: Para otimização de loops e acesso a memória.
    • scf: Para fluxo de controle estruturado (if, for, while).
    • llvm: Um dialeto que representa 1:1 o LLVM IR, servindo como a “ponte” final.

No nosso primeiro exemplo, usamos operações do dialeto func (como func.func e func.return) e do dialeto arith (como arith.constant).

Forma textual: nome_do_dialeto.nome_da_operacao (ex: arith.constant). Nota: Frequentemente, o dialeto func é usado como padrão e o prefixo pode ser omitido, escrevendo apenas func @main.

18.0.2 Operações (Operations)

Tudo no MLIR** é uma operação.**

Isso não pode ser subestimado. module, func, constant, add, return… todas são “operações”.

Uma operação tem a seguinte estrutura geral:

// %resultado = nome_da_operação(%argumentos) {regiões} : (tipos_argumentos) -> (tipos_resultado)
%c42_i32 = arith.constant 42 : i32

No qual temos: * %c42_i32: o(s) nome(s) do(s) valor(es) de resultado. * =: Atribuição (padrão SSA). * arith.constant: o nome da operação (incluindo seu dialeto). * 42 : i32: argumentos da operação. Neste caso, um atributo (o valor 42) e o tipo do resultado.

18.0.3 Valores e SSA

O MLIR usa o formato SSA (Static Single Assignment).

  • Valores são “registradores virtuais” que são definidos exatamente uma vez e podem ser consumidos múltiplas vezes.
  • A sintaxe para um nome de valor é o prefixo % (ex: %valor, %c42_i32, %soma).
  • Valores são o resultado de operações (ex: %c = arith.constant ...) ou argumentos de bloco (ex: argumentos de função).

18.0.4 Tipos (Types)

O MLIR possui um sistema de tipos rico e extensível, que também é definido por dialetos.

  • A sintaxe para tipos é sempre precedida por : (dois pontos).
  • Tipos Nativos (Built-in):
    • i32, i64, i1: Inteiros de largura específica.
    • f32, f64: Ponto flutuante.
  • Tipos de Dialetos:
    • tensor<4x16xf32>: Um tensor (usado por dialetos de ML).
    • memref<128xi32>: Uma referência a um buffer de memória.
    • !llvm.ptr: Um ponteiro do dialeto LLVM.

18.0.5 Atributos (Attributes)

Atributos são “metadados” de tempo de compilação. Eles são constantes e anexados às operações.

  • Diferença de Valores: Um %valor é algo que existe em tempo de execução. Um atributo é algo conhecido pelo compilador.
  • No nosso exemplo, arith.constant 42 : i32:
    • 42 é um Atributo (o valor constante).
    • %c42_i32 é o Valor (o resultado da operação).
  • Atributos também são usados para configurar operações (ex: func @main() attributes { "user" = "professor" }).
  • A sintaxe para atributos nomeados complexos usa # (ex: #meu_mapa = { ... }).

18.0.6 Blocos e Regiões (Blocks and Regions)

Esta é a base do fluxo de controle e do aninhamento.

  • Bloco (Block): Uma sequência de operações que termina com uma operação terminadora (ex: return, br [branch]). Um bloco pode receber argumentos (como em um loop).
  • Região (Region): Um contêiner que agrupa um ou mais blocos.

Uma função (func.func) é uma operação que possui uma região (o corpo da função). O corpo da função, por sua vez, contém um ou mais blocos.

18.1 Anatomia do Nosso Primeiro Exemplo

Vamos revisitar o código com este novo vocabulário:

// 'module' é uma OPERAÇÃO (do dialeto 'builtin').
// Ela contém uma única REGIÃO (o corpo do módulo).
module {

  // 'func.func' é uma OPERAÇÃO (do dialeto 'func').
  // '@main' é um SÍMBOLO (um tipo de atributo).
  // '() -> i32' define a assinatura (tipo) da função.
  // Ela também contém uma única REGIÃO (o corpo da função).
  func.func @main() -> i32 {
    // O corpo da função é um BLOCO (o bloco de entrada).

    // 'arith.constant' é uma OPERAÇÃO (dialeto 'arith').
    // '42' é um ATRIBUTO (valor constante).
    // ': i32' é o TIPO do resultado.
    // '%c42_i32' é o VALOR SSA resultante.
    %c42_i32 = arith.constant 42 : i32

    // 'func.return' é uma OPERAÇÃO terminadora (dialeto 'func').
    // '%c42_i32' é o VALOR que está sendo retornado.
    return %c42_i32 : i32
  } // Fim do Bloco e da Região da função
} // Fim da Região do módulo

18.1.1 Resumo da Sintaxe

  • // ...: Comentário de linha única.
  • @nome: Um símbolo (ex: nome de função).
  • %nome: Um valor SSA (ex: resultado de uma operação).
  • : <tipo>: Anotação de tipo (ex: : i32).
  • { ... }: Delimita uma Região (ex: corpo do module ou func).

18.2 Exercícios Práticos: Aula 1 - Introdução

19 Lista de Exercícios de Introdução ao MLIR

19.1 Módulo 1: Fundamentos de func e arith

19.1.1 Exercício 1: O “Olá, Mundo!”

Objetivo: Criar o módulo MLIR mais simples possível que possa ser compilado e executado.

Tarefa: Escrever um módulo MLIR com uma função @main que não recebe argumentos e retorna o valor inteiro constante 42.

19.1.2 Exercício 2: Função Identidade (Argumentos)

Objetivo: Aprender a definir e usar argumentos de função.

Tarefa: Escreva uma função chamada @identity (identidade) que: 1. Receba um argumento do tipo i32. 2. Retorne esse mesmo argumento.

Dica de Sintaxe: A assinatura da função mudará para func @identity(%arg0: i32) -> i32. O argumento %arg0 já está disponível para você usar dentro da função.

19.1.3 Exercício 3: Aritmética Simples

Objetivo: Usar o dialeto arith para realizar cálculos.

Tarefa: Escreva uma função chamada @add_ten que: 1. Receba um argumento %x do tipo i32. 2. Crie um valor constante 10 do tipo i32. 3. Some %x e o constante 10 usando a operação arith.addi. 4. Retorne o resultado da soma.

Operação de Ajuda: %soma = arith.addi %x, %c10 : i32

19.1.4 Exercício 4: Múltiplos Argumentos e Tipos

Objetivo: Combinar múltiplos argumentos e trabalhar com tipos diferentes (ponto flutuante).

Tarefa: Escreva uma função chamada @multiply_f32 que: 1. Receba dois argumentos, %a e %b, ambos do tipo f32 (ponto flutuante de 32 bits). 2. Multiplique %a por %b usando a operação arith.mulf. 3. Retorne o resultado f32.

Operação de Ajuda: %produto = arith.mulf %a, %b : f32

19.1.5 Exercício 5: Condicionais (scf.if e arith.cmpi)

Objetivo: Aprender a usar o dialeto scf para criar uma expressão condicional if-then-else.

Tarefa: Escreva uma função chamada @max que: 1. Receba dois argumentos, %a e %b, ambos do tipo i32. 2. Compare-os para determinar qual é o maior. 3. Retorne o maior dos dois valores.

Operações de Ajuda: 1. arith.cmpi (Compare Integer): %cond = arith.cmpi "sgt", %a, %b : i32 (onde "sgt" significa “signed greater than”). 2. scf.if (Structured If):

```mlir
%resultado = scf.if %condicao -> (tipo_retorno) {
  scf.yield %valor_then
} else {
  scf.yield %valor_else
}
```
