1. Fundamentos do Módulo 2

No módulo 2, os alunos aprofundam-se na Abstração e no Reconhecimento de Padrões, dois pilares fundamentais do Raciocínio Algorítmico. Este módulo é estruturado em torno da Metodologia DAAD, que inclui as etapas de Decomposição, Abstração, Algoritmização e Depuração.

Objetivos do Módulo
  • Princípios de abstração: identificação de informações essenciais e criação de modelos simplificados
  • Reconhecimento de padrões: identificação de similaridades, tendências e generalizações
  • Abstração na programação: componentes reutilizáveis, funções e estruturas de controle
  • Estruturas de dados simples: vectors, arrays e structs
  • Exercícios avançados de interpretação, visualização de dados e simulação de sistemas
  • Identificação de padrões de código recorrentes para refatoração

Divisão do Trabalho

A partir do exercício H2, os alunos devem ser incentivados a dividir o trabalho em partes para facilitar a implementação e a depuração.

Metodologia
Trabalho em grupos sem controle de versão

Linguagem de Escolha

O professor deve decidir qual linguagem será utilizada durante os Módulos 2 e 3. Python ou C++23 são escolhas recomendadas.

Considerações
Adequação ao nível dos alunos e familiaridade do professor
Importante:

Embora o reconhecimento de padrões não esteja explicitado no acrônimo DAAD, esta competência é um elemento essencial do Raciocínio Algorítmico e da resolução de problemas.

2. Abstração: Reutilização

Aplicação da abstração na programação: criação de componentes reutilizáveis, funções, laços de repetição e estruturas de dados. Utilizando a Linguagem C++ com as abstrações de função, for, while, do while.

1 Exercícios de Revisão

Revisão 1: Maior e Menor Dígito

Dado um número inteiro positivo N (1 ≤ N ≤ 10⁹), encontre o maior e o menor dígito que compõem este número usando apenas operações aritméticas básicas.

Objetivo
Treinar manutenção de estado durante iteração

Revisão 2: Série de Taylor

Implementar calculadora científica que calcule o seno de ângulos usando série de Taylor para sen(x).

Fórmula
sin(x) = x - x³/3! + x⁵/5! - x⁷/7! + ...
Progressão:

Os exercícios de revisão incluem também o cálculo de pi usando série de Leibniz e aproximação fatorial usando fórmula de Stirling, preparando os alunos para conceitos mais avançados.

Revisão 3: Série de Leibniz

Calcular pi usando π/4 = 1 - 1/3 + 1/5 - 1/7 + 1/9 - ... mostrando convergência a cada 1000 iterações.

Meta: Precisão de 6 casas decimais

Revisão 4: Fórmula de Stirling

Aproximar fatoriais usando fórmula de Stirling combinada com série de correção e método babilônico para raiz quadrada.

Implementação: Produtório, somatório e raiz por Newton

3. Reconhecimento de Padrões: Função

O reconhecimento de padrões é uma habilidade essencial no raciocínio algorítmico, permitindo identificar soluções comuns para problemas recorrentes. Foi o reconhecimento de padrões que levou à criação de funções, que são blocos reutilizáveis de código que encapsulam uma lógica específica.

Exercícios com Funções (A-E)

A. Trajetória de Projétil

Canhão dispara projéteis seguindo h(t) = -4.9t² + v₀sin(θ)·t + h₀. Determinar quando o projétil atinge o solo usando série de Taylor para seno.

Conceitos: Equações quadráticas e aproximações trigonométricas

B. Terreno Retangular

Fazendeiro com 120m de cerca para delimitar terreno retangular. Área A = x(60-x). Determinar dimensões para área específica.

Equação: x² - 60x + área = 0

C. Posição de Veículo

Veículo com posição s(t) = s₀ + v₀t + ½at². Calcular instantes em que passa por posição final sf.

Análise: Movimento uniforme vs uniformemente variado

D. Lucro Empresarial

Função lucro L(q) = -2q² + 80q - 300. Determinar quantidade de produção para lucro desejado.

Otimização: Vértice da parábola e pontos críticos

E. Antena Parabólica

Ganho G = -0.1d² + 3d + 5 em função do diâmetro. Calcular diâmetro para ganho mínimo desejado.

Aplicação: Engenharia de telecomunicações
Sequência de Fibonacci na Aplicação:

Exercícios aplicados seguindo a técnica:

  1. Exercício A completamente resolvido pelo professor
  2. Exercício B resolvido pelos alunos (1 problema)
  3. Exercícios C-D resolvidos em duplas (2 problemas)
  4. Exercício E e revisões resolvidos em grupos (3+ problemas)

4. Usando Funções da Biblioteca Padrão

A maioria das linguagens modernas possui funções pré-definidas otimizadas e testadas. Os exercícios exploram o uso de bibliotecas padrão para criar sistemas especialistas.

A2. Casa Lotérica

Sistema para calcular probabilidades de jogos: Mega-Sena, Lotofácil, Quina. Usando funções próprias e da biblioteca padrão.

Funções: calcularFatorial(), calcularCombinacao(), pow(), log()

B2. Sistema Bancário

Sistema para orientar clientes sobre investimentos, calcular juros compostos e tempo para dobrar capital.

Cálculos: Montante final, fator de crescimento, validação

C2. Engenharia Civil

Sistema para calcular parâmetros de rampas de acessibilidade e escadas, validando conformidade com normas.

Trigonometria: sin(), cos(), tan(), atan(), conversões

D2. Engenharia Acústica

Sistema para calcular níveis de ruído, converter intensidades para decibéis e somar múltiplas fontes.

Fórmulas: dB = 10×log₁₀(I/I₀), atenuação por distância

E2. Engenharia Elétrica

Sistema para calcular parâmetros de circuitos trifásicos, eficiência energética e análise de harmônicos.

Conceitos: Potências P, Q, S; fator de potência; THD

F2. Observatório Astronômico

Sistema para calcular posições celestes, distâncias interplanetárias e janelas de lançamento espacial.

Mecânica: Equação de Kepler, coordenadas, órbitas

G2. Laboratório Farmacêutico

Sistema para análise estatística de dados experimentais e modelagem farmacocinética sequencial.

Modelos: C(t) = C₀·e^(-kt), clearance, bioequivalência
Aplicação da Sequência de Fibonacci:

Os exercícios A2-G2 seguem a progressão 1, 1, 2, 3 da Sequência de Fibonacci, com tempo decrescente por problema e complexidade crescente.

5. Decomposição e Abstração: Estruturas de Dados Compostas

Ao completar os exercícios fundamentais, os alunos estão prontos para desafios mais complexos envolvendo estruturas de dados compostas, como arrays, vetores, strings, tuplas e registros.

Estratégia de Divisão de Trabalho

Os exercícios são mais desafiadores e demorados. O professor pode sugerir trabalho em grupos de 4 alunos, dividindo os exercícios em módulos implementáveis separadamente.

Metodologia: Trabalho colaborativo sem sistema de controle de versão, com rotação de papéis entre exercícios.

H2. Biblioteca (Arrays)

Gerenciar empréstimos de livros com array para até 10 registros. Funções: registrar, salvar, ler, buscar empréstimos.

Divisão: 4 alunos, cada um responsável por uma função específica

I2. Tarefas (Vetores)

Organizar tarefas diárias em lista dinâmica com vector. Adicionar, remover e buscar tarefas por palavra-chave.

Estrutura: Vector dinâmico com persistência em arquivo

J2. Loja (Strings)

Registrar produtos vendidos como strings formatadas "nome,preco" em array de até 15 produtos.

Formato: Array de strings com validação e busca

K2. Clube (Structs)

Gerenciar participantes em eventos culturais usando structs (nome, eventos, total pago).

Estrutura: Struct Participante com validação de campos

L2. Sensores (Arrays)

Processar dados de temperatura de sensores municipais. Até 30 temperaturas com estatísticas detalhadas.

Análise: Média, extremos, desvio padrão, tendências

M2. Calculadora (Histórico)

Calculadora científica com histórico completo para auditoria. Array de operações e análise estatística.

Funções: sin(), cos(), tan(), exp(), log10() com histórico

N2. Vendas (Vetores²)

Organizar vendas por categoria usando vetores de vetores. Cada categoria contém lista de valores.

Complexidade: Estruturas bidimensionais dinâmicas
Decomposição Colaborativa:

Cada exercício H2-N2 inclui estratégia detalhada de divisão entre 4 alunos, com interfaces claras para integração final e rotação de responsabilidades.

6. Exercícios para Prática

Os exercícios deste módulo são organizados em progressão de complexidade, combinando conceitos de abstração, reconhecimento de padrões e estruturas de dados compostas.

Exercícios A-E

Reconhecimento de Padrões com Funções
Desenvolvimento de funções para resolver problemas matemáticos e físicos recorrentes.

Conceitos
Equações quadráticas, trigonometria, otimização

Exercícios A2-G2

Funções da Biblioteca Padrão
Sistemas especialistas usando funções otimizadas e testadas das bibliotecas padrão.

Domínios
Matemática, engenharia, ciências, astronomia

Exercícios H2-N2

Estruturas de Dados Compostas
Projetos colaborativos usando arrays, vetores, strings e structs com divisão de trabalho.

Metodologia
Trabalho em grupos de 4 alunos com rotação

Progressão do Módulo

Semanas 4-5: Fundamentos

Exercícios de revisão e introdução a funções. Criação da cognição necessária para abstração e reutilização.

Semanas 6-7: Aplicação

Uso de bibliotecas padrão em sistemas especialistas. Desenvolvimento de projetos com funções matemáticas avançadas.

Semana 8: Integração

Estruturas de dados compostas e trabalho colaborativo. Projetos que combinam todos os conceitos do módulo.

Avaliação e Depuração:

O módulo enfatiza a prática de Depuração através de divisão de trabalho e integração colaborativa, preparando os alunos para projetos mais complexos no Módulo 3.
