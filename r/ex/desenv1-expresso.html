A1 Geração de Relatórios Financeiros

Problema

A contabilidade da empresa "Números Certos" precisa que o Sr. Costa visualize a "tabuada" de um número (multiplicado de 1 a 10).

Objetivo
Imprimir tabuada completa de um número

Solução com Laço

Usar laço de repetição para executar multiplicação de 1 a 10, exibindo cada resultado.

Estrutura
contador de 1 até 10, resultado = numero × contador
# Pseudocódigo para geração de tabuada
ALGORITMO GeracaoTabuadaConciso

ENTRADA:
    numero: inteiro

SAÍDA:
    resultado: inteiro

INÍCIO
    LER numero
    contador ← 1
    
ETIQUETA LOOP:
    resultado ← numero * contador
    ESCREVER numero, " x ", contador, " = ", resultado
    contador ← contador + 1
    SE contador <= 10 ENTÃO IR PARA LOOP
    
FIM

Pseudocódigo

# ALGORITMO SistemaLoginConciso

CONSTANTES:
    USUARIO_VALIDO ← "admin"
    SENHA_VALIDA ← "123"

ENTRADA:
    nomeUsuário: texto
    senha: texto

INÍCIO
    LER nomeUsuário, senha
    loginSucesso ← (nomeUsuário == USUARIO_VALIDO) E (senha == SENHA_VALIDA)
    RETORNAR loginSucesso
FIM

Código C++23

int main() {
    // CONSTANTES
    const std::string USUARIO_VALIDO = "admin";
    const std::string SENHA_VALIDA = "123";
    
    // ENTRADA
    std::string nomeUsuário, senha;
    
    std::cout << "Nome: ";
    std::cin >> nomeUsuário;
    std::cout << "Senha: ";

6. For, While, Do While

Nesta seção, o professor deve escolher se irá continuar com C++23 ou fará a transição para Python. O objetivo é criar o entendimento dos artefatos sintáticos que foram criados para representar laços de repetição: for, while e do while.

Laço FOR

Usado quando se sabe o número exato de iterações que serão realizadas. 

for (int i = 1; i <= 10; i++) {
    std::cout << i << std::endl;
}
Exemplo
Tabuada: multiplicação de 1 a 10

Laço WHILE

Usado quando não sabemos quantas iterações serão necessárias. 

while (true) {
    std::cin >> venda;
    if (venda < 0) break;
    // processar venda
}
Exemplo
Análise de vendas até valor negativo

Laço DO-WHILE

Executa pelo menos uma vez antes de verificar a condição. 

do {
    std::cin >> nota;
    if (nota < 0 || nota > 100) {
        std::cout << "Inválida!";
    }
} while (nota < 0 || nota > 100);
Exemplo
Validação de notas com repetição

Comparação: GOTO vs Estruturas de Controle

Aspecto Com GOTO Com FOR/WHILE/DO-WHILE
Legibilidade Menor, requer rastreamento manual Maior, estrutura clara e intuitiva
Manutenibilidade Difícil de modificar e depurar Fácil de modificar e estender
Propósito Pedagógico Construir cognição sobre repetição Aplicar conhecimento em estruturas avançadas
Indústria Evitado na prática profissional Padrão recomendado universalmente
Estratégia Pedagógica:

O goto é usado inicialmente para que os alunos compreendam como funcionam os laços internamente. Uma vez construída essa cognição, a transição para for, while e do-while torna-se natural e intuitiva.

"Proteção Total": máximo 3 tentativas para senha "seguro123". Decrementar contador, informar tentativas restantes.

Condição: WHILE tentativas > 0 AND senha incorreta

8. Avaliação: Problemas Ad-hoc

Problemas Ad-hoc na Metodologia DAAD

Um problema ad-hoc é um quebra-cabeça que depende de uma observação específica, de uma regra matemática ou de uma simulação cuidadosa, utilizando apenas as ferramentas mais fundamentais da programação.

A dificuldade não está em escrever o código (muitas vezes curto e direto), mas em descobrir a chave que destrava a solução. Como esta disciplina está proposta para os primeiros períodos, os enunciados foram simplificados, transformando-os em problemas de Raciocínio Algorítmico estruturado.
