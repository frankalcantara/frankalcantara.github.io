1. Introdução: Uma Nova Forma de Pensar

Imagine explorar o universo dos computadores não como comandantes que ditam cada passo, mas fornecendo diretrizes gerais e deixando que o computador deduza o caminho. Esta é a essência da Programação Lógica.

"Se vi mais longe foi por estar sobre ombros de gigantes"

Isaac Newton

O Questionamento Fundamental

Em vez de sermos forçados a ordenar cada detalhe do processo de solução, a Programação Lógica permite que declaremos o que queremos e deixemos o computador encontrar como fazer.

Do processo para o problema

O Objetivo Final

Desde os tempos de Gödel, Turing e Church, tudo que queremos é que nossas máquinas sejam capazes de resolver problemas complexos com o mínimo de interferência nossa.

2. Paradigmas de Programação: Imperativo vs Declarativo

A compreensão da diferença fundamental entre os paradigmas de programação é essencial para apreciar o poder da Programação Lógica.

Programação Imperativa

Partimos de uma determinada expressão e seguimos um conjunto de instruções até encontrar o resultado desejado.

Foco no COMO
Define o fluxo de controle
Modifica Estado
Muda o estado da máquina a cada passo
Exemplos
C++, Java, Python

Programação Declarativa

O programador fornece uma descrição lógica ou funcional do que deve ser feito, sem especificar o fluxo de controle.

Foco no O QUE
Descreve o problema, não a solução
Sem Fluxo Explícito
Deixa a implementação para o sistema
Exemplos
SQL, Prolog, Haskell

Programação Lógica: Um Paradigma Declarativo Especial

Na Programação Lógica, um dos paradigmas da Programação Declarativa, usamos a dedução para resolver problemas. Começamos com hipóteses e conjecturas.

Hipóteses

Uma suposição, expressa na forma de proposição, que é acreditada ser verdadeira, mas que ainda não foi provada.

Conjecturas

Uma proposição ou sentença declarativa assumida como verdadeira, mas que ainda não foi formalmente provada ou refutada.

3. Lógica de Primeira Ordem: O Fundamento

Em nossa jornada, percorreremos a Lógica de Primeira Ordem, subdividida em elementos interligados e interdependentes: a Lógica Proposicional e a Lógica Predicativa.

Lógica Proposicional

Um tipo de linguagem matemática, suficientemente rica para expressar os problemas que precisamos resolver e suficientemente simples para que computadores possam lidar com ela.

Característica:
Base sólida e computacionalmente tratável

Lógica Predicativa

A alma da Lógica de Primeira Ordem, também conhecida como Lógica de Predicados, que nos permite fazer sentido do mundo real de forma clara e bela.

Característica:
Expressividade para problemas reais

Exemplos de Conjecturas em Lógica de Primeira Ordem

Exemplo 1: "Todos os números primos são ímpares"

∀x (Primo(x) → Ímpar(x))

Refutação: Esta conjectura pode ser refutada pelo contraexemplo do número 2, que é primo, mas não ímpar.

Exemplo 2: "Todo ser humano é mortal"

∀x (Humano(x) → Mortal(x))

Verificação: Esta conjectura pode ser considerada verdadeira em muitos contextos, mas requer verificação formal para ser aceita como teorema.

A Verdade da Lógica

"A verdade da lógica não abarca opiniões ou contestações. É linda e inquestionável."

Para testar a verdade expressa nessas sentenças, utilizaremos as ferramentas da Lógica de Primeira Ordem, construindo um sistema formal capaz de verificar e deduzir conclusões válidas.

4. Inferência e Dedução: Ferramentas do Detetive

Vamos enfrentar a inferência e a dedução, duas ferramentas fundamentais para extração de conhecimento de declarações lógicas.

A Metáfora do Detetive

A inferência é quase como um detetive que tira conclusões a partir de pistas: teremos algumas verdades (nossas pistas), e precisaremos descobrir outras verdades (consequências diretas das primeiras) para encontrar o que procuramos de forma incontestável.

Processo Dedutivo
Das pistas às conclusões lógicas

Inferência

Processo de derivar conclusões lógicas a partir de premissas conhecidas, seguindo regras de dedução válidas.

Entrada Verdades conhecidas
Processo Regras de dedução
Saída Novas verdades

Dedução

Raciocínio que parte de premissas gerais para chegar a conclusões específicas, garantindo validade lógica.

Exemplo:
1. Todos os homens são mortais
2. Sócrates é homem
3. Logo, Sócrates é mortal

O Objetivo da Automatização

O processo de descobrir a verdade através da inferência e dedução pode ser automatizado, permitindo que máquinas realizem raciocínio lógico complexo.

5. Cláusulas de Horn: O Mapa Simplificador

Nossos mares não serão brandos, mas não nos furtaremos a enfrentar as especificidades da Cláusula de Horn, um conceito que torna todos os problemas expressos em lógica mais fáceis de resolver.

Uma Regra Especial

Uma Cláusula de Horn é como um mapa que, se seguido corretamente, torna o processo de descobrir a verdade mais simples. Muito mais simples, até mesmo passível de automatização.

Característica Principal
Simplificação computacional

Definição Formal

Uma cláusula de Horn é uma cláusula (disjunção de literais) que contém no máximo um literal positivo.

¬A₁ ∨ ¬A₂ ∨ ... ∨ ¬Aₙ ∨ B

Equivalente à regra: A₁ ∧ A₂ ∧ ... ∧ Aₙ → B

Vantagens Computacionais

Decidibilidade
Algoritmos eficientes existem
Resolução Linear
Estratégias de prova simples
Base do Prolog
Implementação prática

Tipos de Cláusulas de Horn

Tipo Forma Interpretação
Fatos B Proposições verdadeiras incondicionalmente
Regras A₁ ∧ A₂ ∧ ... ∧ Aₙ → B Se as condições são verdadeiras, então a conclusão é verdadeira
Consultas ¬A₁ ∨ ¬A₂ ∨ ... ∨ ¬Aₙ Perguntas sobre a verdade de proposições

6. História e Evolução: Sobre Ombros de Gigantes

A Programação Lógica representa uma das pérolas mais reluzentes da evolução humana, construída sobre os trabalhos de gigantes da matemática e computação.

Linha do Tempo dos Gigantes

Kurt Gödel

Estabeleceu os fundamentos da lógica matemática moderna e os teoremas da incompletude, que influenciaram profundamente a teoria da computação.

Alan Turing

Criou o conceito de máquina de Turing e estabeleceu as bases teóricas da computação, incluindo o problema da decidibilidade.

Alonzo Church

Desenvolveu o cálculo lambda e, junto com Turing, estabeleceu a tese de Church-Turing sobre computabilidade.

Isaac Newton

Embora anterior aos outros, sua frase "Se vi mais longe foi por estar sobre ombros de gigantes" captura o espírito científico que guia nossa jornada.

Origens (Anos 1970)

O termo Programação Lógica surge como evolução dos esforços em:

Prova de Teoremas: Automação de demonstrações matemáticas
Inteligência Artificial: Simulação do raciocínio humano
Lógica Computacional: Aplicação prática da lógica formal

Prolog: A Implementação

O surgimento do Prolog (Programming Logic) materializa a esperança de usar lógica como linguagem de programação.

Objetivo
Máquinas capazes de raciocinar como o homem
Resultado
Linguagem declarativa baseada em lógica
