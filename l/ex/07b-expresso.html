1. Introdução ao Paradigma Lógico

"O Prolog representa uma mudança de paradigma em relação às linguagens imperativas. Em vez de fornecer instruções sobre como executar uma tarefa, especificamos o que é verdade sobre um domínio."

Programming in Logic

A mágica por trás da capacidade do Prolog de responder a perguntas complexas não é arbitrária. É o resultado direto da interação sistemática de três processos distintos que formam seu mecanismo de inferência:

Unificação

Algoritmo de correspondência de padrões que serve como a operação computacional primária.

Papel
Torna dois termos sintaticamente idênticos

Resolução SLD

Procedimento de prova formal que fornece a base lógica para a inferência.

Função
Constrói provas por refutação

Backtracking

Estratégia de busca em profundidade com mecanismo robusto de tentativa e erro.

Estratégia
Explora sistematicamente alternativas

Paradigma Declarativo

Um programa em Prolog é uma especificação executável de um subconjunto da Lógica de Primeira Ordem, permitindo que os programadores se concentrem na descrição da lógica e das relações do problema.

Separação Fundamental: Dados (fatos) • Lógica (regras) • Controle (mecanismo de inferência)

2. A Arquitetura de um Programa Prolog

Todos os programas Prolog são estruturados a partir da base de conhecimento (knowledge base), um repositório estruturado de asserções lógicas que descrevem um domínio específico.

Base de Conhecimento

O texto do programa não é uma sequência de instruções, mas sim uma declaração estática de verdades. Executar um programa é, na verdade, o ato de submeter uma consulta à base de conhecimento.

Fatos

Dados brutos do domínio, verdades incondicionais

Regras

Conhecimento condicional, implicações lógicas

Consultas

Perguntas ao sistema, objetivos a provar

3. Termos: Os Dados do Prolog

Estrutura Unificada de Dados

Em Prolog, toda a representação de dados é feita através de uma única estrutura: o termo. Não existem tipos de dados distintos no sentido convencional.

Átomos

Constantes que representam objetos, propriedades ou relações. Começam com letra minúscula.

socrates, gosta, vermelho

Variáveis

Espaços reservados para termos desconhecidos. Começam com letra maiúscula ou underscore.

X, Pessoa, _Nome, _

Números

Literais numéricos, inteiros e ponto flutuante.

42, 3.14, -17

Estruturas

Termos compostos com functor e argumentos. Principal meio de criar dados complexos.

functor(arg1, arg2, ...)
pessoa(nome(joao), idade(25))

Listas: Estruturas Especiais

Listas são estruturas com sintaxe especial. A lista [a, b, c] é internamente representada como:

'.'(a, '.'(b, '.'(c, [])))

A notação [H|T] decompõe uma lista em cabeça (H) e cauda (T).

4. Cláusulas: A Lógica do Prolog

As cláusulas constituem as unidades fundamentais de conhecimento em um programa Prolog. Elas codificam tanto conhecimento factual quanto inferencial sobre um domínio específico.

Fatos: Conhecimento Incondicional

Fatos são cláusulas unitárias que declaram verdades incondicionais. Correspondem a cláusulas de Horn unitárias da forma $P(t_1, t_2, \cdots, t_n)$.

Exemplos de Fatos

% Predicados unários - propriedades
mortal(socrates).
mamifero(gato).
ave(canario).

% Predicados binários - relações
pai(joao, pedro).
maior_que(5, 3).
capital(brasil, brasilia).

% Predicados n-ários - relações complexas
distancia(sao_paulo, rio_janeiro, 400).
nasceu(socrates, atenas, 470).
Interpretação Formal: Cada fato P(a1, ..., an) é interpretado como "é verdadeiro que P(a1, ..., an)" no modelo mínimo de Herbrand.

Regras: Conhecimento Condicional

Regras estabelecem verdades condicionais através de implicações lógicas. Correspondem a cláusulas de Horn definidas:

$H \leftarrow B_1 \land B_2 \land ... \land B_n$

Exemplos de Regras

% Regra simples: ∀X (mamifero(X) → animal(X))
animal(X) :- mamifero(X).

% Regra com múltiplas condições
pode_voar(X) :- 
    ave(X), 
    \+ pinguim(X), 
    \+ avestruz(X).

% Regra recursiva para ancestralidade
ancestral(X, Y) :- pai(X, Y).
ancestral(X, Z) :- pai(X, Y), ancestral(Y, Z).

Quantificação e Estrutura Lógica

Quantificação Universal

Variáveis em regras são universalmente quantificadas:
animal(X) :- mamifero(X)
significa: $\forall X (mamifero(X) \rightarrow animal(X))$

Disjunção por Múltiplas Cláusulas

Múltiplas cláusulas com mesma cabeça expressam disjunção:
voa(X) :- ave(X).
voa(X) :- inseto_voador(X).

5. Consultas: Interrogando a Base de Conhecimento

Natureza Formal das Consultas

Consultas são o mecanismo através do qual fazemos perguntas à base de conhecimento. Correspondem a cláusulas de Horn negativas da forma:

$\leftarrow B_1 \land B_2 \land ... \land B_n$

O mecanismo de inferência tenta encontrar uma refutação da negação da consulta, implementando prova por contradição.

Consultas Ground

Sem variáveis, perguntam sobre veracidade de fato específico.

?- pai(joao, pedro).
Resposta: true/false

Com Variáveis Livres

Solicitam instanciações que tornam a afirmação verdadeira.

?- pai(X, pedro).
X = joao

Consultas Conjuntivas

Múltiplas condições que devem ser satisfeitas.

?- pai(X, Y), mae(Z, Y).
Encontra X, Y, Z

Processo de Resolução de Consultas

1
Decomposição: Múltiplos objetivos tratados como conjunção
2
Busca por Correspondência: Sistema busca cláusulas unificáveis
3
Aplicação de Regras: Objetivo substituído pelos objetivos do corpo
4
Construção de Provas: Processo continua recursivamente

6. A Hipótese do Mundo Fechado

Princípio Fundamental

CWA - Closed World Assumption: Tudo o que não pode ser provado verdadeiro a partir da base de conhecimento é assumido como falso.

Na lógica clássica, a ausência de informação não permite conclusões. O Prolog adota a hipótese contrária: a ignorância implica falsidade.

Exemplo Demonstrativo

% Base de conhecimento
ave(canario).
ave(papagaio).
voa(canario).

% Consultas demonstrando a CWA
?- ave(canario).   % true - pode ser provado
?- ave(cachorro).  % false - não pode ser provado
?- voa(papagaio).  % false - não pode ser provado

Consequências Práticas

  • Comportamento determinístico
  • Negação por falha (\+)
  • Sistema completo para o domínio

Limitações

  • Monotonicidade quebrada
  • Colapso falso/desconhecido
  • Sensível a mudanças na base

Negação por Falha

O operador \+ não representa negação lógica clássica, mas sim "falha em provar": 

solteiro(X) :- pessoa(X), \+ casado(X).

% Se casado(joao) não pode ser provado,
% então solteiro(joao) é verdadeiro

7. O Coração do Mecanismo: Unificação

"A unificação é a operação fundamental que impulsiona toda a computação em Prolog."

Mais do que simples correspondência de padrões, é um algoritmo sofisticado para tornar dois termos sintaticamente idênticos, resolvendo equações simbólicas. Processo bidirecional que pode instanciar variáveis em qualquer termo.

Algoritmo de Unificação

1
Constantes idênticas: Unificação tem sucesso
2
Variável não instanciada: Liga-se ao outro termo
3
Duas variáveis livres: Tornam-se co-referenciadas
4
Estruturas: Mesmo functor/aridade + argumentos unificam
5
Outros casos: Unificação falha
Termo 1 Termo 2 Unifica? MGU (Unificador Mais Geral)
gato(tom) gato(tom) Sim {} (substituição vazia)
gato(X) gato(tom) Sim X = tom
pai(X, Y) pai(joao, maria) Sim X = joao, Y = maria
f(X, X) f(a, b) Não Falha (a ≠ b)
lista([a,b]) lista([X|Y]) Sim X = a, Y = [b]

Unificador Mais Geral (MGU)

O MGU é a substituição que faz o menor número de compromissos, deixando variáveis tão livres quanto possível. Maximiza opções para futuras unificações.

Occurs Check

Verificação omitida por eficiência. Previne termos cíclicos como X = f(X). Disponível via unify_with_occurs_check/2.

8. Resolução SLD: O Mecanismo de Inferência Lógico

Prova por Refutação

Para provar que uma consulta Q é verdadeira, o mecanismo assume temporariamente que ela é falsa (¬Q) e tenta derivar uma contradição lógica.

SLD significa:
  • Selective - Seleção do objetivo mais à esquerda
  • Linear - Sequência linear de passos de prova
  • Definite - Opera sobre cláusulas de Horn definidas

Algoritmo de Resolução SLD

1
Seleção: Escolhe o objetivo mais à esquerda
2
Pesquisa: Busca cláusula unificável de cima para baixo
3
Fato encontrado: Remove objetivo, aplica MGU
4
Regra encontrada: Substitui objetivo pelo corpo
5
Sem unificação: Objetivo falha
6
Sucesso: Lista de objetivos vazia

Exemplo: Árvore de Busca SLD

% Base de conhecimento
pai(joao, maria).
pai(joao, pedro).
pai(pedro, ana).

progenitor(X, Y) :- pai(X, Y).
progenitor(X, Y) :- mae(X, Y).

ancestral(X, Y) :- progenitor(X, Y).
ancestral(X, Y) :- progenitor(X, Z), ancestral(Z, Y).

% Consulta: ?- ancestral(joao, ana).
Processo de Resolução:
  1. Tenta primeira cláusula: ancestral(X,Y) :- progenitor(X,Y)
  2. Busca progenitor(joao, ana) - FALHA
  3. Backtrack, tenta segunda cláusula
  4. Encontra progenitor(joao, maria) - SUCESSO
  5. Resolve ancestral(maria, ana) recursivamente
  6. Encontra caminho completo - SUCESSO

9. Navegando no Espaço de Soluções: Backtracking

Busca em Profundidade

O Prolog emprega uma estratégia de busca em profundidade (DFS), seguindo uma única linha de raciocínio o mais profundamente possível antes de considerar alternativas.

Vantagens
  • Eficiente em memória
  • Comportamento previsível
  • Determinístico
Limitações
  • Sistema incompleto
  • Pode prender em loops
  • Sensível à ordem

Mecanismo de Backtracking

1
Ponto de Escolha: Criado quando existem alternativas
2
Execução para Frente: Segue caminho escolhido
3
Ativação: Quando objetivo falha
4
Retorno: Volta ao ponto de escolha mais recente
5
Desfazer Ligações: Desinstancia variáveis
6
Nova Tentativa: Explora próxima alternativa

Exemplo: Busca de Caminhos com Backtracking

edge(a,b). 
edge(a,c). 
edge(b,d). 
edge(c,d). 
edge(d,e).

path(X,Y) :- path(X,Y,[]).

% Caso base: aresta direta
path(X,Y,_) :- edge(X,Y).

% Caso recursivo: caminho via Z
path(X,Y,V) :-  
    \+ member(X,V),  
    edge(X,Z),  
    path(Z,Y,[X|V]).

% Consulta: ?- path(a, e).
Rastreamento do Backtracking:

1. Tenta caminho via b: a→b→d→e ✓
2. Se pedir mais soluções (;), backtrack
3. Tenta caminho via c: a→c→d→e ✓
4. Explora sistematicamente todas as alternativas

10. Estruturas de Dados como Lente para a Mecânica

A manipulação de estruturas de dados em Prolog demonstra a aplicação prática de unificação e recursão. O código não descreve como manipular, mas o que define a estrutura.

Listas: A Estrutura Fundamental

A lista é a única estrutura com sintaxe própria em Prolog, definida recursivamente:

Lista vazia: []
Lista não-vazia: [Head|Tail]
Predicado member/2
% Caso base: elemento é a cabeça
member(X, [X|_]).

% Caso recursivo: procura na cauda
member(X, [_|T]) :- member(X, T).

Pilhas: Ajuste Natural

Estrutura LIFO trivialmente implementada usando a cabeça como topo: 

% Pilha vazia
is_empty([]).

% Adiciona ao topo
push(Elem, Stack, [Elem|Stack]).

% Remove do topo
pop(Elem, [Elem|Stack], Stack).

% Vê o topo sem remover
peek(Elem, [Elem|_]).
Nota: Operações não modificam a pilha original; geram nova versão (imutabilidade).

Árvores Binárias de Busca

Estrutura recursiva ideal para Prolog: tree(Esq, Valor, Dir) ou nil 

% Busca em árvore
lookup(Value, tree(_, Value, _)).
lookup(Value, tree(Left, NodeVal, _)) :-
    Value < NodeVal,
    lookup(Value, Left).
lookup(Value, tree(_, NodeVal, Right)) :-
    Value > NodeVal,
    lookup(Value, Right).

% Inserção em árvore
insert(Value, nil, tree(nil, Value, nil)).
insert(Value, tree(L, V, R), tree(NL, V, R)) :-
    Value < V,
    insert(Value, L, NL).
insert(Value, tree(L, V, R), tree(L, V, NR)) :-
    Value > V,
    insert(Value, R, NR).

11. Controle Manual: O Operador de Corte (!)

Concessão ao Pragmatismo

O corte (!) representa uma concessão do modelo declarativo ao pragmatismo procedural, permitindo intervir diretamente no comportamento de busca.

Efeitos do Corte:
  • Descarta outras cláusulas do predicado abaixo
  • Descarta pontos de escolha à esquerda
  • Compromete com escolhas feitas

Cortes Verdes

Não alteram significado declarativo, apenas melhoram eficiência. 

max(X, Y, X) :- X >= Y, !.
max(X, Y, Y) :- X < Y.

Remove redundância sem alterar resultados.

Cortes Vermelhos

Alteram significado do programa, tornam correção dependente da ordem. 

ifte(C, T, _) :- C, !, T.
ifte(_, _, E) :- E.

Segunda cláusula só funciona protegida pelo corte.

Negação como Falha

A implementação do operador \+ usa corte vermelho com fail: 

not(Goal) :- call(Goal), !, fail.
not(_).

% Se Goal tem sucesso → corte → fail → not(Goal) falha
% Se Goal falha → backtrack → segunda cláusula → not(Goal) sucesso

Inverte efetivamente sucesso/falha do objetivo original.
