1. Lógica de Primeira Ordem

A Lógica de Primeira Ordem é uma estrutura básica da ciência da computação e da programação. Ela nos permite discursar e raciocinar com precisão sobre os elementos.

Equilíbrio Fundamental

Esta restrição não é um defeito, mas sim um equilíbrio cuidadoso entre poder expressivo e simplicidade computacional.

Capacidade
Formular grande variedade de problemas

Pedra Fundamental

Uma forma poderosa e útil de olhar para o universo, não tão complicada que seja hermética, mas suficientemente complexa para descobrir mistérios matemáticos.

Objetivo
Resolver problemas práticos

Capacidades e Limitações

✓ Podemos fazer

  • Afirmações sobre todo um grupo
  • Afirmações sobre elemento particular
  • Raciocínio com precisão

⚠ Limitações clássicas

  • Afirmações diretas sobre predicados
  • Afirmações diretas sobre funções
  • Extensões: Lógica de Segunda Ordem

2. Alfabeto e Sintaxe da Lógica de Primeira Ordem

A Lógica de Primeira Ordem é uma linguagem criada a partir de um alfabeto Σ, de um conjunto de axiomas e de um conjunto de regras de inferência.

1. Variáveis e Constantes

Símbolos do alfabeto latino e pontuação

Variáveis: u, v, w, x, y, z
Constantes: a, b, c, d, e
Pontuação: ( )

2. Funções e Predicados

Símbolos para operações e relações

Funções: f, g, h, i
Predicados: P, Q, R, S
Ou strings: MaiorQue, IgualA

3. Operadores

Conectivos lógicos tradicionais

Negação: ¬
Conjunção: ∧
Disjunção: ∨
Implicação: →
Equivalência: ↔

4. Quantificadores

Tradição matemática para quantificação

Existencial: ∃
Universal: ∀
Significado: "existe" e "para todo"

5. Fórmulas Bem Formadas

Expressões válidas da linguagem

Representação: P, Q, R, S
Valores: TRUE, FALSE
Propriedade: Só podem ser verdadeiras ou falsas

6. Termos e Átomos

Elementos básicos das fórmulas

Termos: variáveis, constantes, funções
Átomos: predicados aplicados a termos
Exemplo: P(x), MaiorQue(a,b)

Fórmulas Bem Formadas (FBF)

"Em Lógica de Primeira Ordem, uma Fórmula Bem Formada é uma expressão que só pode ser verdadeira ou falsa."

As FBF são compostas de símbolos que representam quantificadores, variáveis, constantes, predicados, e conectivos lógicos, seguindo regras sintáticas, gramaticais e semânticas da linguagem lógica.

Aprendizado
Aprender lógica é aprender esta linguagem

3. Precedência de Operadores e Uso de Parênteses

A regra sintática mais importante é a precedência das operações, uma espécie de receita indexada que deve ser seguida à letra.

Ordem de Precedência

¬, ∀, ∃, ∧, ∨, →, ↔

Maior precedência: ¬ (negação)   |   Menor precedência: ↔ (equivalência)

✓ Com Parênteses (Explícito)

(∀x(∃y (P(x,y)→Q(x))))
Sem ambiguidade, fácil de interpretar

⚠ Sem Parênteses (Ambíguo)

∀x ∃y (P(x,y)→Q(x))
Pode permitir outras interpretações

Regras para Termos

1.
Variável
Uma variável x é um termo em si
2.
Constante
Uma constante a é um termo em si
3.
Função
Se f é função de termos (t₁, ... tₙ) então f(t₁, ... tₙ) é um termo
4.
Valores Lógicos
T/TRUE para verdadeiro, F/FALSE para falso

Regras para Átomos e FBF

1.
Átomo Predicativo
P(t₁, ... tₙ) é uma FBF básica
2.
Conectivos
Se P, Q são FBF então ¬P, P∧Q, P∨Q, P→Q, P↔Q são FBF
3.
Quantificadores
Se P é FBF e x variável então ∃x P(x) e ∀x P(x) são FBF

Definição da Linguagem

"Podemos definir a linguagem da Lógica de Primeira Ordem como o conjunto de todas as Fórmulas Bem Formadas criadas a partir dos campos de estudo da Lógica Proposicional e da Lógica de Predicados."

Termos e átomos interligados em uma teia, em que cada termo ou átomo é como uma ilha de verdade. As operações lógicas são as pontes que conectam essas ilhas, permitindo-nos construir as estruturas mais complexas da razão.

4. Fundamentos da Lógica Proposicional

Esse sistema, também chamado de álgebra booleana, fundamental para o desenvolvimento da computação, é uma verdadeira tapeçaria de possibilidades.

"Na Lógica Proposicional, declarações atômicas, que só podem ter valores verdadeiro (TRUE) ou falso (FALSE), são entrelaçadas em declarações compostas cuja veracidade depende dos valores de verdade das declarações atômicas que as compõem quando sujeitas aos conectivos."

Proposições Atômicas

Representamos por letras como P, Q, R, S, ou usando índices como X₁, X₂, etc. Cada proposição é um fato fundamental e indivisível.

Exemplos
"A chuva cai", "O sol brilha"

Literais

Um literal é uma variável proposicional (P ou Q) ou sua negação (¬P ou ¬Q). São as unidades básicas da Lógica Proposicional.

Combinação
Conectivos lógicos formam fórmulas complexas

Estrutura das Fórmulas

Fórmulas Atômicas

Proposições básicas que não podem ser decompostas em partes mais simples usando conectivos lógicos.

Fórmulas Compostas

Um operador principal (conectivo) liga duas fórmulas atômicas ou duas fórmulas compostas.

Construção: As declarações atômicas e compostas são costuradas por conectivos para produzir declarações compostas, cujo valor de verdade depende dos valores de verdade das declarações componentes.

5. Análise dos Conectivos Lógicos

Os conectivos lógicos são os elementos fundamentais que nos permitem combinar proposições atômicas para formar expressões mais complexas.

Tabela 1 - Tabela Verdade dos operadores básicos da Lógica Proposicional

∨ (Disjunção)

Também chamado de ou inclusivo, é verdadeiro quando pelo menos um dos termos é verdadeiro.

Diferença:
∨ difere de ⊕ (ou exclusivo)

∧ (Conjunção)

Verdadeiro apenas quando ambos os termos são verdadeiros. Corresponde ao "e" lógico.

Exemplo:
"o céu está azul e a grama é verde"

¬ (Negação)

Operador unário que inverte o valor-verdade de uma proposição.

Propriedade:
TRUE quando proposição é FALSE e vice-versa

→ (Implicação)

Falso apenas quando o antecedente é verdadeiro e o consequente é falso. Não implica causalidade.

Importante:
Lógico, não causal

↔ (Bicondicional)

Equivale a ambos os componentes terem o mesmo valor-verdade. Corresponde a "se e somente se".

Condição:
Verdadeiras ou falsas simultaneamente

Aridade dos Operadores

No Argumentos Aridade Exemplos
1 Unário P(x), ¬P
2 Binário P ∨ Q, P ∧ Q
Observação importante: Se tivermos quatro termos diferentes, em vez de dois, teremos 2⁴ = 16 linhas. Se todos os resultados forem TRUE, temos uma tautologia; se todos forem FALSE, uma contradição.

6. Equivalências Lógicas

As equivalências lógicas permitem validar Fórmulas Bem Formadas sem recorrer a tabelas-verdade, simplificando expressões e facilitando provas lógicas.

Tautologia

Uma fórmula que é sempre verdadeira, não importa os valores dados às variáveis. Verdades universais no domínio do problema.

Aplicação:
Simplificar expressões e fazer inferências válidas automaticamente

Contradição

Uma fórmula que é sempre falsa, independentemente dos valores das variáveis. Mostra inconsistências ou impossibilidades lógicas.

Benefício:
Evita custo de tentar provar algo logicamente impossível

As equivalências surgem naturalmente enquanto trabalhamos. Vejamos exemplos práticos de simplificação usando apenas as equivalências da Tabela 3.

Nível Básico

Exemplo 1: P ∧ (Q ∨ (P ∧ R))

1
P ∧ (Q ∨ (P ∧ R))
Fórmula original
2
≡ (P ∧ Q) ∨ (P ∧ (P ∧ R))

8. Sistema Dedutivo da Lógica Proposicional

O sistema dedutivo é um conjunto de regras e princípios que nos permite derivar conclusões a partir de premissas, fundamental para a construção de argumentos lógicos válidos.

Importância Histórica

"A Lógica Proposicional é essencial para entendermos o mundo. É a base de argumentos sólidos e da avaliação de proposições."

Claude Shannon

Em sua dissertação, usou a álgebra booleana para simplificar circuitos de controle, criando a base dos sistemas digitais modernos.

George Boole

Sua álgebra sustenta o design de circuitos eletrônicos e a construção dos computadores modernos.

Operações Fundamentais: As operações básicas AND, OR, NOT tornaram-se os blocos fundamentais dos sistemas digitais, formando o núcleo de toda nossa civilização digital.

9. Regras de Inferência

Regras de inferência são esquemas que proporcionam a estrutura para derivações lógicas, formando a base dos sistemas de tomada de decisão computacional.

Notação das Regras

Uma regra de inferência será representada por:

P₁, P₂, ..., Pₙ
──────────
C
P₁, P₂, ..., Pₙ ⊢ C
Significado: O conjunto {P₁, P₂, ..., Pₙ} é o contexto (antecedente), e C é a conclusão (consequente). Se as premissas do contexto são verdadeiras, então a conclusão também será verdadeira.
Modus Ponens
P → Q
P
─────
Q

Se temos uma implicação P→Q e sabemos que P é verdadeiro, então podemos concluir que Q também é verdadeiro.

Exemplo Natural:
• Se chover, a rua ficará molhada
• Está chovendo
• Logo, a rua ficará molhada
Modus Tollens
P → Q
¬Q
─────
¬P

Se temos uma implicação P→Q e sabemos que Q é falso, então podemos concluir que P também é falso.

Exemplo Natural:
• Se uma pessoa tem 18 anos ou mais, então pode votar
• Maria não pode votar
• Logo, Maria não tem 18 anos ou mais
Adjunção
P
Q
─────
P ∧ Q

Se P e Q são verdadeiros, então P ∧ Q é verdadeiro. Permite combinar duas afirmações em uma conjunção.

Exemplo Natural:
• O céu está azul
• Os pássaros estão cantando
• Logo, o céu está azul e os pássaros estão cantando
Simplificação
P ∧ Q
─────
P

Se P ∧ Q é verdadeiro, então P (ou Q) é verdadeiro. Permite extrair componentes individuais de uma conjunção.

Exemplo Natural:
• O céu está azul e os pássaros estão cantando
• Logo, o céu está azul
Modus Tollendo Ponens
P ∨ Q
¬P
─────
Q

Dada uma disjunção P∨Q, se ¬P então Q. Permite inferir uma disjunção a partir da negação da outra.

Exemplo Natural:
• Ou o céu está azul ou a grama é roxa
• A grama não é roxa
• Logo, o céu está azul
Dupla Negação
¬¬P
─────
P

A negação de uma negação é equivalente à afirmação original. Importante para simplificar expressões lógicas.

Exemplo Natural:
• Não é verdade que Maria não está feliz
• Logo, Maria está feliz

Resumo das Regras de Inferência

Regra Descrição Fórmula
Modus Ponens Se P → Q e P são verdadeiros, então Q também é verdadeiro P, P → Q / Q
Modus Tollens Se P → Q e ¬Q são verdadeiros, então ¬P também é verdadeiro ¬Q, P → Q / ¬P
Dupla Negação A negação de uma negação é equivalente à afirmação original ¬¬P / P
Adição Se P é verdadeiro, então P ∨ Q também é verdadeiro P / P ∨ Q
Adjunção Se P e Q são verdadeiros, então P ∧ Q é verdadeiro P, Q / P ∧ Q
Simplificação Se P ∧ Q é verdadeiro, então P (ou Q) é verdadeiro P ∧ Q / P
Bicondicionalidade Se P ↔ Q, então P → Q e Q → P são verdadeiros P ↔ Q / P → Q, Q → P
Fundamento das Provas: As regras de inferência são o alicerce da lógica dedutiva e das provas matemáticas. Elas permitem que raciocínios complexos sejam divididos em passos simples, com cada passo sendo justificado pela aplicação de uma regra de inferência.

10. Análise Semântica de Fórmulas

A análise semântica estuda o significado e as propriedades de verdade das expressões lógicas, examinando quando e por que essas fórmulas são verdadeiras ou falsas.

Sintaxe vs Semântica

Sintaxe

Preocupa-se com a estrutura formal das fórmulas, as regras de formação e a organização dos símbolos.

Semântica

Examina quando e por que essas fórmulas são verdadeiras ou falsas, focando no significado.

Satisfatível

R é satisfatível se sua Tabela Verdade contém pelo menos uma linha verdadeira.

Exemplo: P ∧ Q
Verdadeiro quando P=TRUE e Q=TRUE

Insatisfatível

R é insatisfatível se sua Tabela Verdade contém apenas linhas falsas.

Exemplo: P ∧ ¬P
Sempre falso (contradição)

Falsificável

R é falsificável se sua Tabela Verdade contém pelo menos uma linha falsa.

Exemplo: P ∧ Q
Falso quando P=FALSE ou Q=FALSE

Válida (Tautologia)

R é válida se sua Tabela Verdade contém apenas linhas verdadeiras.

Exemplo: P ∨ ¬P
Sempre verdadeiro (tautologia)

Exemplo: Análise de P ∧ Q

P Q P ∧ Q Classificação
FALSE FALSE FALSE Linha falsa
FALSE TRUE FALSE Linha falsa
TRUE FALSE FALSE Linha falsa
TRUE TRUE TRUE Linha verdadeira
Conclusão: P ∧ Q é satisfatível (tem pelo menos uma linha verdadeira) e falsificável (tem pelo menos uma linha falsa), mas não é uma tautologia.

Propriedades Importantes

✓ Relações Válidas

  • Se A é válida, então A é satisfatível
  • Se A é insatisfatível, então A é falsificável

📋 Explicação

Uma fórmula válida (tautologia) é satisfatível porque tem pelo menos uma linha verdadeira. Uma fórmula insatisfatível (contradição) é falsificável porque tem pelo menos uma linha falsa.

Aplicações Práticas da Classificação

1. Análise de Argumentos

Se uma argumentação pode ser representada por uma fórmula insatisfatível, então o argumento é inválido ou inconsistente. Usado em lógica e filosofia para analisar validade.

2. Prova de Teoremas

Podemos mostrar que a negação da fórmula é insatisfatível, demonstrando que a fórmula original é uma tautologia.

3. Simplificação de Fórmulas

Se parte de uma fórmula é tautologia, podemos simplificar removendo essa parte. Se é contradição, a fórmula inteira é insatisfatível.

4. Construção de Argumentos

Garantir que argumentos são válidos verificando se são representados por fórmulas satisfatíveis (mas não tautologias).

Uso em Programação Lógica:
• Unificação e resolução fazem deduções usando tautologias
• Satisfatibilidade: se obtivermos contradição, as premissas são insatisfatíveis
• Reconhecer contradições evita custo de tentar provar algo logicamente impossível
